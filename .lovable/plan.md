
# Исправление порядка и подсветки в нижнем меню

## Изменения

**Файл: `src/components/BottomNavigation.tsx`**

### 1. Новый порядок вкладок
Текущий: Прогресс, Задания, Следующее, Теория, Главная
Нужный: Главная, Задания, Следующее задание, Прогресс, Теория

```tsx
const tabs = [
  { title: "Главная", icon: Home, path: "/" },
  { title: "Задания", icon: ClipboardList, path: "/tasks" },
  { title: "Следующее", icon: PlayCircle, path: "next" },
  { title: "Прогресс", icon: TrendingUp, path: "/my-progress" },
  { title: "Теория", icon: BookOpen, path: "/webinar-records" },
];
```

### 2. Подсветка иконки "Следующее задание"
Сейчас при нажатии на "Следующее" происходит навигация на страницу задания, но `selected` сбрасывается через `useEffect`, потому что `path: "next"` не совпадает ни с одним URL. Иконка не подсвечивается.

Исправление: в `handleChange`, когда нажата кнопка "Следующее", устанавливать `selected` на индекс этой кнопки и не сбрасывать его в `useEffect`, если текущий путь -- это страница задания (начинается с `/task/` или `/skill-assignments/`). Таким образом после перехода к заданию иконка "Следующее" останется подсвеченной.

Простое решение -- в `useEffect` проверять: если текущий путь не соответствует ни одной вкладке, оставлять предыдущее значение `selected` (а не сбрасывать в `null`):

```tsx
useEffect(() => {
  const currentPath = location.pathname;
  const index = tabs.findIndex(tab => 
    tab.path !== "next" && currentPath === tab.path
  );
  if (index !== -1) {
    setSelected(index);
  }
  // Если не найден -- не сбрасываем, чтобы "Следующее" оставалось подсвеченным
}, [location.pathname]);
```

## Итого: один файл, две правки
- Переупорядочить массив `tabs`
- Убрать сброс `selected` в `null` при навигации на страницу задания
